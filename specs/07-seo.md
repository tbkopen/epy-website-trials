# 07 — SEO

## Overview

SEO is critical for a blog-first site. The goal is to rank for the topics covered in posts, driving organic readers who then become course prospects. This file covers the technical SEO layer — meta tags, structured data, sitemap, and performance signals.

---

## Core SEO Configuration

`_config.yml` settings for `jekyll-seo-tag`:
```yaml
title: "Elastropy"
description: "[One-sentence description of what Elastropy is about — TBD]"
url: "https://elastropy.com"
author:
  name: "[Your Name — TBD]"
  email: "[optional]"
  twitter: "@[your-handle — TBD]"
logo: /assets/images/logo.png
social:
  name: "Elastropy"
  links:
    - https://twitter.com/[your-handle]
    - https://github.com/[your-handle]
    - https://linkedin.com/in/[your-handle]
```

`{% seo %}` in `_includes/head.html` generates all standard meta tags automatically.

---

## Meta Tag Checklist

`jekyll-seo-tag` handles all of these automatically from frontmatter + config:

| Tag | Source |
|-----|--------|
| `<title>` | `page.title | site.title` |
| `meta description` | `page.description` or `page.excerpt` |
| `canonical URL` | `page.url` + `site.url` |
| `og:title` | `page.title` |
| `og:description` | `page.description` or `page.excerpt` |
| `og:image` | `page.image` or `site.logo` |
| `og:type` | `article` for posts, `website` for other pages |
| `twitter:card` | `summary_large_image` when `page.image` set |
| `twitter:creator` | `page.author` → `site.twitter` |

**Per-post**: set `excerpt:` and optionally `image:` in frontmatter for accurate OG tags.

## Structured Data (JSON-LD)

Add structured data in `_includes/structured-data.html`, conditionally included by layout.

### BlogPosting (for all posts)
```liquid
<script type="application/ld+json">
{
  "@context": "https://schema.org",
  "@type": "BlogPosting",
  "headline": {{ page.title | jsonify }},
  "description": {{ page.excerpt | strip_html | jsonify }},
  "url": "{{ site.url }}{{ page.url }}",
  "datePublished": "{{ page.date | date_to_xmlschema }}",
  "dateModified": "{{ page.last_modified_at | default: page.date | date_to_xmlschema }}",
  "author": {
    "@type": "Person",
    "name": {{ site.author.name | jsonify }},
    "url": "{{ site.url }}/about/"
  },
  "publisher": {
    "@type": "Person",
    "name": {{ site.author.name | jsonify }},
    "logo": {
      "@type": "ImageObject",
      "url": "{{ site.url }}{{ site.logo }}"
    }
  }{% if page.image %},
  "image": "{{ site.url }}{{ page.image }}"{% endif %}
}
</script>
```

### Course (for course pages)
```liquid
<script type="application/ld+json">
{
  "@context": "https://schema.org",
  "@type": "Course",
  "name": {{ page.title | jsonify }},
  "description": {{ page.description | jsonify }},
  "url": "{{ site.url }}{{ page.url }}",
  "provider": {
    "@type": "Person",
    "name": {{ site.author.name | jsonify }},
    "url": "{{ site.url }}"
  },
  "offers": {
    "@type": "Offer",
    "price": "{{ page.price }}",
    "priceCurrency": "{{ page.currency | default: 'USD' }}"
  }
}
</script>
```

### BreadcrumbList
```liquid
{% assign crumbs = page.breadcrumbs %}
<script type="application/ld+json">
{
  "@context": "https://schema.org",
  "@type": "BreadcrumbList",
  "itemListElement": [
    { "@type": "ListItem", "position": 1, "name": "Home", "item": "{{ site.url }}/" },
    { "@type": "ListItem", "position": 2, "name": "Blog", "item": "{{ site.url }}/blog/" },
    { "@type": "ListItem", "position": 3, "name": {{ page.title | jsonify }}, "item": "{{ site.url }}{{ page.url }}" }
  ]

</script>
```

### Person (on About page and home)


## Sitemap

Auto-generated by `jekyll-sitemap` plugin at `/sitemap.xml`.

Exclude pages from sitemap via frontmatter:
```yaml
sitemap: false
```

Exclude by default:
- `/thanks/` pages
- `/404.html`

Submit sitemap to Google Search Console and Bing Webmaster Tools after launch.

`robots.txt` at root (create manually or via plugin):
```

Sitemap: https://elastropy.com/sitemap.xml
```

## Open Graph Images

Posts should have an `image:` in frontmatter for rich previews when shared on social.

**Dimensions**: 1200 × 630px (standard OG image size)
**Location**: `/assets/images/posts/YYYY-MM-DD-slug.jpg`
**Format**: JPG or WebP

If no image is set, `jekyll-seo-tag` falls back to `site.logo`. Create a default OG image with the site name for this fallback.

Optional: generate post OG images programmatically (e.g., using a Cloudinary template or a build-time script) to avoid creating images manually.

## Performance (Core Web Vitals)

Poor Core Web Vitals hurt rankings. Target these scores:

| Metric | Target | How |
|--------|--------|-----|
| LCP (Largest Contentful Paint) | < 2.5s | Preload hero image, serve via CDN, avoid large images |
| INP (Interaction to Next Paint) | < 200ms | Minimal JS; defer Remark42 and KaTeX |
| CLS (Cumulative Layout Shift) | < 0.1 | Set explicit width/height on images; don't inject above-fold elements with JS |

Key optimizations:
- Serve images as WebP with `srcset` for responsive sizes
- Lazy-load all below-fold images (`loading="lazy"`)
- Preconnect to third-party origins: KaTeX CDN, Remark42 host, email platform
- Inline critical CSS (small; Jekyll makes this easy)
- All `<script>` tags use `defer` or `async`

## Content SEO Checklist (per post)

- [ ] Title includes primary keyword naturally (50–60 chars)
- [ ] `excerpt:` is filled in (150–160 chars, includes keyword)
- [ ] H2/H3 headings use related keywords (not keyword-stuffed)
- [ ] At least one internal link to a related post or course
- [ ] `image:` set with descriptive `alt` text on the `<img>` tag
- [ ] Slug is descriptive and concise (no stop words)
- [ ] `last_modified_at:` updated when post is significantly revised

---

## URL Permanence

Never change post URLs after publishing without setting up a redirect. Use `jekyll-redirect-from`:
```yaml
redirect_from:
  - /old/path/
```

Or set up HTTP 301 redirects at the hosting level (Netlify `_redirects` file or Cloudflare Rules).
